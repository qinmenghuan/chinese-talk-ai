# Discovery 页面系统设计

## 1. 目标

为 `Discovery` 页面增加“对话主题查询 + 卡片列表展示 + 分页加载 + 跳转练习”能力。

核心目标：

- 用户可以按条件搜索对话主题
- 用户可以浏览分页卡片列表
- 用户可以点击卡片进入 `practice` 页面开始练习

## 2. 页面职责

`Discovery` 页面只负责“查找并选择主题”，不承担练习逻辑。

页面包含 2 个主要区域：

- 查询区
- 主题卡片列表区

## 3. 查询条件

支持以下筛选项：

- 关键词：按主题标题和描述做模糊查询
- 难度：下拉筛选
- 主题类型：下拉筛选

查询原则：

- 任一条件可单独使用
- 多个条件可组合使用
- 修改条件后，列表重置到第一页

## 4. 列表展示

主题列表继续使用卡片布局，并抽成公共组件，便于首页和 Discovery 复用。

卡片建议包含：

- 封面图
- 主题标题
- 主题描述
- 难度
- 类型
- 进入练习按钮

交互要求：

- 点击整张卡片或按钮，都可进入 `practice`
- 跳转时带上 `scenarioId`、默认 `roleId`、`mode`

## 5. 分页方案

列表采用分页加载，一页 20 条。

建议方案：

- 前端维护 `page`、`pageSize`、`hasMore`
- 首次进入加载第一页
- 查询条件变化时重新拉取第一页
- 点击“加载更多”或滚动触底加载下一页

返回结构建议：

- `items`
- `page`
- `pageSize`
- `total`
- `hasMore`

## 6. 接口设计

建议新增主题查询接口：

- `GET /scenarios`

支持参数：

- `keyword`
- `difficulty`
- `type`
- `page`
- `pageSize`

说明：

- `keyword` 同时匹配 `title` 和 `subtitle/description`
- `pageSize` 固定默认 20
- 返回结果直接供 Discovery 页面渲染

## 7. 公共组件设计

建议抽取公共组件：

- `ScenarioCard`

组件职责：

- 只负责展示单个主题卡片
- 接收主题数据和跳转参数
- 不内置页面级查询和分页逻辑

这样可以让：

- 首页场景卡片复用
- Discovery 列表复用
- 后续推荐列表也可复用

## 8. 响应式布局

卡片列表需要根据屏幕大小自适应排列。

建议布局：

- 移动端：1 列
- 平板：2 列
- 小桌面：3 列
- 大桌面：4 列

要求：

- 卡片高度尽量统一
- 图片比例稳定
- 筛选区在移动端也能正常折行

## 9. 页面状态

页面需要覆盖以下状态：

- 初始加载
- 查询结果为空
- 正常展示
- 加载更多中
- 接口失败

对应处理：

- 加载中：骨架屏或 loading
- 空结果：显示“没有匹配主题”
- 失败：显示错误提示和重试入口

## 10. 最小落地方案

按最少改动实现时，建议顺序：

1. 新增 Discovery 页面路由
2. 新增主题查询接口
3. 抽取 `ScenarioCard` 公共组件
4. 实现查询表单
5. 实现分页卡片列表
6. 接通跳转到 `practice`

## 11. 验收标准

- 能根据关键词、难度、类型搜索主题
- 列表支持分页加载，每页 20 条
- 卡片布局自适应不同屏幕宽度
- 点击主题卡片可进入 `practice` 页面
- 卡片组件已抽取为可复用公共组件
