# 作用域： apps/web,apps/api,apps/admin

# 目标：登录功能增加邮箱密码登录和注册功能

# 需求

- web端
  - 页面
    - 首页右上角有登录和注册入口，登录成功后后则只显示账号
  - 登录
    - 点击登录按钮,弹框登录框，显示账号密码登录或者google账号登录
  - 注册
    - 使用邮箱和密码注册，注册后跳转到首页，需要重新登录
  - 设置页面
    - 表单的label和控件在一行内布局，形成共通规范
    - 点击账号下拉选项-设置按钮，跳转到单独的设置页面
    - 页面可以设置Display name，level，Learning goal，Preferred Doubao voice

- admin端
  - users 模块可以查看注册后的所有用户，
  - 可以修改基础信息，可以停用/启用相关账号
  - 停用后的账号，web端不能登录，强行登录给出友好的提示

# 限制：不要看其他的docs

- 登录过的用户
  - 可以访问任何页面
- 未登录的用户
  - 可以访问首页和discover页面
  - 点击对话主题卡片或者discover页面的对话卡片或者Practice，History页面，toast提示英文的请先登录

# 规范：

- 遵守项目中的代码和UI规范
