# 首页页面设计文档

## 1. 背景

本设计文档基于 `docs/current-task/current-task.md`，作用域限定为 `apps/web` 首页体验优化。

本次目标不是新增功能，而是收敛首页表达方式：减少营销式说明和重复功能介绍，让用户第一眼看到清晰的站点导航、登录入口和可直接进入练习的对话主题。

## 2. 设计目标

- 首页整体风格保持简约、干净、轻量。
- 保持当前 `packages/design-tokens` 视觉规范，继续使用白色画布、深色文字、主色按钮、柔和边框和轻量阴影。
- Header 改为明确的左中右结构：左侧品牌、中央导航、右侧账号操作。
- 主内容减少大段文案和解释型模块，重点展示对话主题卡片。
- 对话主题以精美图片作为主要视觉入口，点击整张卡片即可进入练习页。
- 主题标签显示难度，不显示角色名。
- 不影响 `apps/admin`，不改动管理台页面和后端接口。

## 3. 页面信息架构

首页建议保留两层结构：

1. Header
2. Main

Main 内部建议只保留核心内容：

1. 简短首页引导区
2. 对话主题网格

移除或弱化当前首页中偏说明型的模块，例如功能卡片、长 CTA 区块、重复解释语句。用户不需要先读完产品介绍才开始练习，页面应把“选择一个对话主题”作为主动作。

## 4. Header 设计

### 4.1 布局

Header 使用三栏结构：

| 区域 | 内容          | 说明                                   |
| ---- | ------------- | -------------------------------------- |
| 左侧 | Logo + 产品名 | 根据中文口语练习主题设计品牌识别       |
| 中部 | 当前导航      | 保持现有导航项与认证门禁逻辑           |
| 右侧 | 登录状态操作  | 未登录显示登录、注册；已登录只显示账号 |

桌面端 header 高度继续使用当前接近 `80px` 的高度，内容区沿用 `max-w-7xl` 和左右页边距。

移动端如果空间不足，优先保留 Logo 与右侧登录入口，导航可维持当前已有的响应式隐藏策略。

### 4.2 Logo

Logo 应贴合“实时中文口语练习”主题，但保持简约：

- 图形可以使用“对话气泡 + 声波”或“中文字形 + 麦克风”的组合。
- 不使用复杂插画，不引入多色渐变。
- 主色使用 `--color-primary`，文字使用 `--color-ink`。
- 产品名可继续使用 `Learn Chinese AI`，也可在图形中加入轻量中文识别，例如 `中` 字符号。

Logo 区域需要是可点击链接，点击返回首页。

### 4.3 中部导航

中部导航保持当前信息结构：

| 导航     | 目标         | 说明         |
| -------- | ------------ | ------------ |
| Home     | `/`          | 首页         |
| Discover | `/discovery` | 发现主题     |
| Practice | `/practice`  | 登录后可进入 |
| History  | `/history`   | 登录后可进入 |

导航交互规则保持现有逻辑：未登录点击受保护导航时，触发登录弹框或认证流程。

视觉上应继续使用 pill 导航，但减少过强装饰。当前激活项可保留主色背景，非激活项使用纯文本或轻量 hover 背景。

### 4.4 右侧账号区

未登录状态：

- 只显示 `Login` 和 `Register`。
- `Login` 使用白底描边按钮。
- `Register` 使用主色按钮。
- 移除与本需求无关的图标按钮和装饰按钮，例如收藏、语言、管理台入口、教学口号等。

已登录状态：

- 只显示登录账号。
- 推荐显示 `displayName`，空间不足时可显示邮箱前缀。
- 账号入口可以保留点击后展开菜单，但 header 默认可见内容只保留账号本身。
- 菜单中可继续包含 `Settings` 和 `Sign out`，但不在 header 常驻展示额外操作。

## 5. Main 设计

### 5.1 首页引导区

首页引导区应短而明确，不再使用大面积营销 hero。

建议内容：

- 一句短标题，表达“选择真实场景，开始中文对话”。
- 一句短说明，不超过一行到两行。
- 不再使用搜索 pill、功能标签组、复杂左右分栏说明。

标题示例：

```text
Choose a scene. Start speaking Chinese.
```

说明示例：

```text
Pick a real-life conversation and jump straight into practice.
```

视觉规则：

- 使用白色背景。
- 标题不超过当前设计 token 的 display 层级。
- 文案区域宽度控制在较窄范围，避免页面变成营销长文。
- 引导区下方直接进入主题网格。

### 5.2 对话主题网格

对话主题是首页主内容，应成为页面视觉重心。

桌面端建议使用 4 列网格，平板使用 2 列，移动端使用 1 列。卡片间距保持轻量，避免页面过于松散。

每张卡片结构：

| 区域     | 内容                                 | 说明                                          |
| -------- | ------------------------------------ | --------------------------------------------- |
| 图片     | `scenario.cover`                     | 主视觉，图片占卡片主要高度                    |
| 顶部标签 | 难度                                 | 显示 `Beginner` / `Intermediate` / `Advanced` |
| 标题     | `scenario.title`                     | 保持清晰短标题                                |
| 辅助信息 | 可选 `scenario.type` 或极短 subtitle | 文案尽量少                                    |

卡片不显示角色名，不显示底部箭头，不显示 `Start this practice` 文案。

### 5.3 卡片点击行为

整张主题卡片都是点击区域：

- 已登录：跳转到对应练习页。
- 未登录：触发登录弹框，并保留目标练习页作为登录后的返回路径。

目标路径继续使用当前 `buildPracticeHref()` 逻辑：

```text
/practice?scenarioId={scenario.id}&roleId={scenario.defaultRoleId}&mode={scenario.mode}
```

卡片 hover 可以保留轻微图片放大或阴影变化，但不增加额外操作按钮。

### 5.4 图片策略

首页视觉重点来自主题图片，因此图片需要更大、更稳定：

- 卡片图片建议使用固定比例，例如 `4:3` 或 `3:2`。
- 图片使用 `object-cover`，保持裁切美观。
- 标题和标签不要遮挡图片主体。
- 难度标签可置于图片左上角，使用白底半透明或纯白 pill，保证可读性。

当前 mock 数据中的 Unsplash 图片可以继续使用。后续若接入真实场景数据，需要保证每个主题都有高质量封面图。

## 6. 视觉规范

继续沿用 `packages/design-tokens`：

| 类型     | Token                               | 用途                    |
| -------- | ----------------------------------- | ----------------------- |
| 背景     | `--color-canvas`                    | 页面主背景              |
| 文本     | `--color-ink`                       | 标题、主要导航          |
| 正文     | `--color-body`                      | 短说明                  |
| 弱文本   | `--color-muted`                     | 类型、次要标签          |
| 主色     | `--color-primary`                   | 注册按钮、激活导航      |
| 边框     | `--color-hairline-soft`             | Header、卡片边线        |
| 卡片圆角 | `--radius-card`                     | 主题卡片                |
| 按钮圆角 | `--radius-pill` / `--radius-button` | 导航和账号按钮          |
| 阴影     | `--shadow-float`                    | 只用于 hover 或轻量浮层 |

整体视觉应接近“白底、图片主导、少量主色点缀”。不要新增大面积背景色、复杂渐变、装饰图形或大段品牌文案。

## 7. 响应式设计

### 7.1 桌面端

- Header 三栏完整显示。
- Logo 左对齐，导航居中，账号操作右对齐。
- 主题网格 4 列。
- 引导区与主题网格垂直排列。

### 7.2 平板端

- Header 保持 Logo、导航、账号区，但减少间距。
- 主题网格 2 列。
- 卡片图片比例保持一致。

### 7.3 移动端

- Header 优先显示 Logo 和登录/账号入口。
- 导航可沿用当前隐藏策略。
- 主题网格 1 列。
- 卡片点击区域保持整卡可点，触摸目标足够大。
- 文案尽量短，避免首屏被说明文字占满。

## 8. 状态与交互

### 8.1 未登录

- Header 右侧显示 `Login`、`Register`。
- 点击受保护导航或主题卡片，打开登录弹框。
- 登录后返回原目标页。

### 8.2 已登录

- Header 右侧只显示账号。
- 点击主题卡片直接进入练习页。
- 账号菜单可保留设置和退出，但默认不展示额外按钮。

### 8.3 Loading

认证状态加载时，右侧按钮可以保持当前 `Checking...` 或使用轻量占位。

首页主题卡片使用当前本地 mock 数据时不需要额外 loading 状态；后续若改为接口数据，再补充 skeleton。

## 9. 建议调整范围

后续实现时，建议只涉及以下文件：

| 文件                                        | 调整内容                                 |
| ------------------------------------------- | ---------------------------------------- |
| `apps/web/app/layout.tsx`                   | Header 左侧 logo 与整体左右结构          |
| `apps/web/components/HeaderAuthActions.tsx` | 右侧登录态展示精简                       |
| `apps/web/components/ScenarioCard.tsx`      | 主题卡片整卡点击、难度标签、移除底部箭头 |
| `apps/web/app/page.tsx`                     | 首页结构收敛，突出主题网格               |

不建议调整：

- `apps/admin`
- 后端认证接口
- 练习页流程
- 历史页和报告页
- 共享类型结构

## 10. 验收标准

- Header 是明确的左中右结构。
- 左侧 Logo 与中文口语练习主题相关，视觉简洁。
- 中部导航项保持当前信息结构与认证门禁。
- 未登录时右侧只显示登录、注册。
- 已登录时右侧只显示账号入口。
- 首页主内容减少说明文字，主题卡片成为视觉重点。
- 主题卡片显示难度，不显示角色名。
- 主题卡片不显示底部箭头引导。
- 点击主题卡片可进入对应练习页；未登录时先触发登录。
- 页面仍符合 `packages/design-tokens` 的 UI 规范。
- 不影响 `apps/admin` 和其他业务页面。

## 11. 实施备注

本设计文档仅定义页面调整方向和验收口径。当前阶段不改动代码。

后续进入实现时，应先调整首页和组件结构，再运行至少以下验证：

```bash
pnpm lint
pnpm typecheck
```

如涉及实际页面检查，再启动 `pnpm dev` 验证桌面与移动端视觉效果。
