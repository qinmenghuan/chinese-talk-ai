# /users 页面简洁设计文档

## 1. 目标

将 `apps/admin` 的 `/users` 页面从当前“卡片列表 + 右侧详情编辑”改为标准管理台交互：

- 顶部查询区
- 中部标准 Table 列表
- 行内操作按钮
- 编辑弹框
- 抽取可复用的管理端公共组件

本次范围只覆盖 `/users` 页面及其复用组件设计，不包含其他管理页改造。

## 2. 当前问题

当前 `/users` 页面存在以下问题：

- 用户数据以卡片列表展示，不符合管理台高密度列表场景
- 查询条件只有关键字，缺少注册时间筛选
- 编辑区域固定占据右侧，不是标准的弹框式编辑
- 启用/停用、编辑、列表容器、表单都直接写在页面里，复用性差

## 3. 目标交互

### 3.1 页面结构

`/users` 页面拆成三段：

1. 页面标题区
2. 查询筛选区
3. 用户列表区

### 3.2 查询区

查询区使用一行或两行标准筛选栏，包含：

- 名称或邮箱：单个输入框
- 注册时间：开始日期、结束日期
- 查询按钮
- 重置按钮

交互规则：

- 点击查询后刷新列表并回到第一页
- 点击重置后恢复默认筛选并重新加载
- 注册时间为空时表示不过滤

## 4. 列表设计

### 4.1 展示方式

用户列表改为标准 Table 组件，建议列如下：

- 用户名称
- 邮箱
- 学习水平
- 学习目标
- 偏好音色
- 状态
- 注册时间
- 最近登录时间
- 操作

### 4.2 状态展示

- `active` 显示为“启用”
- `disabled` 显示为“停用”
- 状态使用 Badge 或轻量状态标签

### 4.3 行内操作

操作列包含两个按钮：

- 启用/停用
- 编辑

交互规则：

- 当前为启用状态时显示“停用”
- 当前为停用状态时显示“启用”
- 点击后可直接执行，建议加二次确认
- 点击“编辑”打开弹框

## 5. 编辑弹框设计

### 5.1 弹框内容

编辑弹框包含：

- 显示名称
- 水平
- 学习目标
- 喜欢的豆包 AI 音色

### 5.2 弹框交互

- 点击行内“编辑”打开
- 打开时带入当前用户数据
- 点击取消关闭，不提交
- 点击保存提交修改
- 保存成功后：
  - 关闭弹框
  - 刷新列表
  - 如当前页存在该用户，列表内容同步更新

## 6. 公共组件抽取

本次要求把 `/users` 页面中的列表、按钮、弹框、表单抽成管理端公共组件。

### 6.1 建议组件目录

建议在 `apps/admin/src/components/admin/` 下抽取：

- `admin-page-toolbar`
  - 管理页筛选栏容器
- `admin-data-table`
  - 通用表格容器
- `admin-row-actions`
  - 行操作按钮组
- `admin-modal`
  - 管理端标准弹框
- `admin-form-field`
  - 表单项包装

### 6.2 `/users` 业务组件

建议在 `apps/admin/src/features/users/` 下放业务组件：

- `users-page`
- `users-filter-form`
- `users-table`
- `user-status-action`
- `user-edit-dialog`
- `user-edit-form`

原则：

- 通用视觉骨架放 `components/admin`
- 用户业务逻辑放 `features/users`

## 7. 状态管理建议

`/users` 页面建议保留页面级状态管理，不引入额外状态库。

页面状态至少包括：

- 查询条件
- 列表数据
- 分页信息
- loading 状态
- 当前编辑用户
- 弹框开关状态
- 提交中的状态

## 8. 接口设计补充

为支持“名称或邮箱 + 注册时间”查询，后端 `GET /api/admin/users` 建议扩展参数：

- `keyword`
- `createdFrom`
- `createdTo`
- `page`
- `pageSize`

返回结构保持现有分页格式即可。

## 9. 实施顺序

建议实施顺序：

1. 抽管理端通用 Table / Modal / Toolbar / FormField / RowActions 组件
2. 重构 `/users` 页面为 `features/users`
3. 接入注册时间筛选
4. 把启用停用改成行内操作
5. 把编辑改成弹框表单
6. 联调列表刷新与局部更新

## 10. 验收口径

本次 `/users` 页面完成后，验收标准为：

- 用户数据通过标准 Table 展示
- 查询条件支持“名称或邮箱”和“注册时间”
- 行内操作支持启用/停用
- 行内操作支持编辑
- 编辑通过弹框完成
- 列表、操作按钮、弹框、表单已抽为管理端公共组件
